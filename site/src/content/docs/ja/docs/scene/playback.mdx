## 再生モード

inboil には 2 つの再生モードがあります：

| モード | 動作 |
|--------|------|
| **Loop** | 現在のパターンをリピート再生（デフォルト） |
| **Scene** | ルートノードからシーングラフをたどる |

**▶** をタップすると Loop モードで再生。**▶ を長押し**（またはモード切替を使用）で Scene 再生を開始します。

## Scene 再生フロー

<PlaygroundSceneView client:visible />

### ステップごとの流れ

1. **ルートノードからスタート** — ドットでマークされたノード
2. **パターンを再生** — ノードが参照するパターンを最初から最後まで再生
3. **デコレーターを適用** — Transpose、Tempo、Repeat、FX、Automation が再生中に適用
4. **リピート** — Repeat デコレーターがあれば、パターンを N 回ループ
5. **次のエッジを選択** — ノードに出力エッジがあれば、ウェイトに基づいてランダムに 1 つ選択
6. **次のノードへ移動** — 選択先のノードで再生を継続
7. **ターミナルで停止** — 出力エッジがなければ、再生は終了

### エッジの選択

分岐点（複数の出力エッジを持つノード）では、エンジンは**ウェイト付きランダム選択**を使います：

```
エッジ A のウェイト：3
エッジ B のウェイト：1
エッジ C のウェイト：1
合計：5

確率 A = 3/5 = 60%
確率 B = 1/5 = 20%
確率 C = 1/5 = 20%
```

これにより**ジェネレーティブなアレンジメント**が生まれます — 同じシーンでも毎回異なる展開になります。

### グラフ内のループ

エッジがサイクルを形成している場合（A → B → C → A）、再生はそれらのノードを無限にループします。これは最も一般的なシーン構成で、バリエーションを持ちながらループする一連のパターンです。

Scene 再生を停止するには、**Space** を押すか **■** をタップします。

## ツーカーソルモデル

inboil は 2 つの独立したカーソルを維持しています：

| カーソル | 制御元 | 目的 |
|---------|--------|------|
| **ユーザーカーソル**（`ui.currentPattern`） | Matrix/Scene でパターンをタップ | 編集中のパターンを表示 |
| **再生カーソル**（`playback.playingPattern`） | シーンエンジン | 現在再生中のパターンを制御 |

Scene 再生中、再生カーソルはグラフを自動的に進みます。ユーザーカーソルは元の位置に留まるので、再生を中断することなくパターンを編集できます。

## テンポの変更

再生が **Tempo** デコレーター付きのノードに入ると：
- テンポが即座に変更（クロスフェードなし）
- そのノードを離れると、テンポは前の値に戻る
- 次のノードにも Tempo デコレーターがあれば、その値が適用

テンポの変更はノード単位であり、グローバルではありません — プロジェクトのテンポを恒久的に変えることはありません。

## Scene 再生中のパフォーマンス

パフォーマンスボタン（**FILL**、**REV**、**BRK**）は Scene 再生中も使えます。現在再生中のパターンにリアルタイムで作用し、デコレーターの上に重ねて適用されます。
