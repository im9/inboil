## 全般

### INBOIL とは？
ブラウザベースのグルーブボックス兼 DAW です。ブラウザ上で完全に動作し、ダウンロード、アカウント登録、プラグインは一切不要。アプリを開いたらすぐにビートメイキングを始められます。

### 対応ブラウザは？
INBOIL は **Web Audio API** と **AudioWorklet** をサポートするモダンブラウザが必要です：
- Chrome 66+（推奨）
- Firefox 76+
- Safari 14.1+
- Edge 79+

モバイルブラウザでも動作しますが、インターフェースはデスクトップ向けに最適化されています。

### 作業内容は保存される？
はい — プロジェクトは編集のたびにブラウザのローカルストレージ（IndexedDB）に**自動保存**されます。ただし、データは**このブラウザ内のみ**に存在します。**EXPORT** で JSON バックアップをダウンロードすれば、どこでもインポートできます。

### オフラインで使える？
初回読み込み後はオフラインでも動作します（PWA です）。ただし、ドキュメントサイトにはインターネット接続が必要です。

## サウンド・オーディオ

### 音が出ない場合は？
1. 再生が開始されているか確認（**Space** を押す）
2. トラックがミュートされていないか確認（**M** ボタン）
3. ブラウザの音量と出力デバイスを確認
4. 一部のブラウザはページを操作するまでオーディオをブロックします — まずどこかをクリックしてください

### オーディオ出力を録音できる？
はい — System パネルの **REC ●** ボタンを使います。マスター出力の WAV ファイルを録音します。REC を押して開始、もう一度押して停止しダウンロードします。

### 外部 MIDI コントローラーは使える？
はい — USB または BLE MIDI キーボードを接続し、**SYSTEM → MIDI** で設定します。INBOIL はノート入力、ベロシティ、チャンネルルーティングに対応しています。

### Export はどんなフォーマットを出力する？
プロジェクトデータ（全パターン、シーン、設定）は **JSON** 形式。**MIDI** エクスポートは標準的な .mid ファイルを生成。**WAV** 録音はリアルタイムでオーディオ出力をキャプチャします。

## パターンとシーケンス

### パターンはいくつ作れる？
1 プロジェクトにつき 100 パターン。

### トラックごとに異なるステップ数を設定できる？
はい — 各トラックは **2〜64** の独立したステップ数を持てます。ステップ数の違いがポリリズムを生み出します。

### パーステップロックとは？
特定の 1 ステップだけサウンドパラメータを変更する機能です。Dock で **LOCK ON** を有効にし、ステップを選択してからノブを回します。そのステップだけ独自のサウンド設定を持ち、パターンの残りは元のサウンドを維持します。

## シーングラフ

### Loop モードと Scene モードの違いは？
**Loop** は現在のパターンをリピート再生します。**Scene** はルートノードからシーングラフをたどり、エッジに沿ってバリエーションのある長いアレンジメントを作ります。

### シーンを無限ループさせられる？
はい — エッジがサイクルを形成していれば（A → B → C → A）、再生はそれらのノードを無限にループします。ウェイト付きエッジを使えば、まったく同じ展開にはならないバリエーションが生まれます。

### ジェネレーティブノードとは？
保存されたパターンを再生する代わりに、アルゴリズムで音楽コンテンツを生成するノードです。利用可能なエンジン：Turing Machine、Quantizer、Tonnetz。

## データとプライバシー

### データはどこに保存される？
すべてのデータはブラウザのローカル **IndexedDB** に保存されます。サーバーには一切送信されません。アカウント、クラウド同期、アナリティクスはありません。

### 作業のバックアップ方法は？
System パネルの **EXPORT** を使って、プロジェクト全体を含む JSON ファイルをダウンロードします。後から **IMPORT** で読み込めます — 別のブラウザでも可能です。

### 初期状態に戻すには？
**SYSTEM → NEW** で新しい空のプロジェクトを作成するか、Help パネルの **FACTORY RESET** ですべてをクリアしてデフォルトに戻します。
